# Multi-Platform Publish

Publish and syndicate content across platforms — Blogger, X/Twitter, LinkedIn,
Medium, email newsletters — with platform-appropriate formatting, scheduling,
and analytics tracking from a single content source.

## What problem this solves
Writing once and manually reformatting for each platform wastes time and creates
inconsistencies. This skill takes one piece of content and adapts it for every
platform — right length, right format, right tone, right hashtags — with
scheduling and performance tracking.

## When to use
When publishing any new content. When maintaining consistent presence across
platforms. When the audience is fragmented across channels.

## Input
Source content (blog post, article, announcement), target platforms, posting schedule.

## Output
Platform-adapted content variants with scheduling and analytics tracking.

## Steps
1. Start with canonical content: blog post, article, or announcement
2. Adapt for each platform:
   - **Blogger:** Full article with SEO metadata, images, internal links
   - **X/Twitter:** Thread (5-8 tweets), key insights, 2-3 hashtags, link to full post
   - **LinkedIn:** Professional summary (150-300 words), 3-5 bullet points, 3 hashtags
   - **Medium:** Canonical URL import or republish with platform-specific intro
   - **Email:** Subject line, preview text, 1-paragraph summary, CTA button
3. Schedule posts: staggered across days, optimal times per platform
4. Cross-link: X post → blog, LinkedIn post → blog, blog → newsletter signup
5. Track analytics: views, engagement, clicks, conversions per platform
6. Maintain content calendar to plan and visualize publishing rhythm

## Rules
- Blog post is the canonical source — every other platform links back to it
- Adapt, don't duplicate — each platform gets content formatted for THAT audience
- X threads: 5-8 tweets, hook in tweet 1, key insight in tweet 3, CTA in last tweet
- LinkedIn: professional tone, 3-5 bullet points, no more than 3 hashtags
- Publishing schedule: Blogger → X (same day) → LinkedIn (next day) → Email (day 3)
