# Training Module

Design modular training courses — structured learning paths with modules,
lessons, labs, and assessments — from single workshops to full certification
programs.

## What problem this solves
Isolated labs don't build comprehensive skills. A training module sequences
learning: concepts → demonstrations → practice → assessment → certification.
Each module builds on the previous, forming a coherent learning journey.

## When to use
When designing a course or certification. When organizing existing labs into
a curriculum. When The Data Professor wants to teach a topic systematically.

## Input
Topic, target audience, duration, delivery format, existing materials.

## Output
A modular course design with learning path, module descriptions, lesson plans,
lab integration, and assessment strategy.

## Steps
1. Define target audience and entry requirements: what should students already know?
2. Design the learning path: modules in sequence, each building on the previous
3. For each module:
   - Learning objectives (3-5 measurable outcomes)
   - Content: concepts + demonstrations
   - Practice: labs or exercises
   - Assessment: knowledge check or practical test
   - Duration: estimated time to complete
4. Structure: 3-5 modules per course, 2-4 hours per module
5. Integrate labs from lab-scenario-design skill
6. Add certification criteria: passing all assessments with 80%+
7. Test with pilot students, gather feedback, iterate

## Rules
- Module sequence must build progressively — no jumps in complexity
- Each module: objectives → content → practice → assessment
- 3-5 modules per course; 2-4 hours per module
- Prerequisites stated explicitly at course and module level
- Assessments must verify learning objectives — not just completion
- Certification requires 80%+ on all assessments
- Coordinate with Eunice (overall education) and lab-scenario-design (labs)
