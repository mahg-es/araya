# Curriculum Planning

Design complete learning curricula — multi-course programs with learning
paths, prerequisites, certifications, and career tracks — for academic or
professional education programs.

## What problem this solves
Individual courses don't make a program. Curriculum planning sequences multiple
courses into a coherent learning journey — from fundamentals to specialization
— with clear milestones, prerequisites, and career outcomes.

## When to use
When designing a multi-course program (Big Data certification, AI engineering
track). When expanding from individual courses to a full curriculum. When The
Data Professor wants to build a comprehensive educational program.

## Input
Domain, target roles, existing courses, industry requirements, certification goals.

## Output
A curriculum design with learning tracks, course sequences, prerequisites,
milestones, and certification paths.

## Steps
1. Define target roles and career outcomes: what jobs will graduates qualify for?
2. Map competency model: what skills, knowledge, and abilities does each role require?
3. Design learning tracks: Foundations → Core → Specialization → Capstone
4. For each course: learning objectives, prerequisites, duration, assessment method
5. Define milestones: certification levels (Associate, Professional, Expert)
6. Sequence courses: prerequisites enforced, progressive complexity
7. Set program duration: full-time, part-time, self-paced options

## Rules
- Tracks: Foundations → Core → Specialization → Capstone
- Each course has clear prerequisites — enforce them
- Certification levels: Associate (foundations), Professional (core), Expert (specialization)
- Industry alignment: curriculum must match job requirements — not just academic interest
- Regular review: update curriculum annually based on industry changes
- Part-time and self-paced tracks for working professionals
- Coordinate with Eunice (education), training-module (courses), and lab-scenario-design (labs)
