# Project Planning

Facilitate project planning — charters, roadmaps, milestones, and resource
plans — transforming ideas into structured, trackable projects with clear
scope, timeline, and success criteria.

## What problem this solves
Projects that start without a plan finish without a result. Project planning
forces clarity upfront: what are we building, for whom, by when, with what?
The resulting charter and roadmap become the project's constitution — changing
deliberately, not accidentally.

## When to use
At project inception. When scope is unclear. When stakeholders need alignment.
When The Data Professor says "let's build X."

## Input
Project idea/vision, constraints, available resources, stakeholder needs.

## Output
A project charter with vision, scope, milestones, resource plan, risk assessment,
and success criteria.

## Steps
1. Define project charter: vision, objectives, scope, constraints, stakeholders
2. Design roadmap: major milestones on a timeline (now, next, later)
3. Identify resources: which agents, what tools, what budget?
4. Assess risks: what could go wrong? (use pm-risk skill)
5. Define success criteria: how do we know when we're done?
6. Present to The Data Professor for approval before work begins
7. Review and update at major milestones

## Rules
- Charter before execution — planning is cheap; rework is expensive
- Scope must have clear boundaries: what's included AND excluded
- Milestones: meaningful achievements, not just calendar dates
- Success criteria must be measurable — "users are happy" is not measurable
- Stakeholders identified upfront — no surprise dependencies
- Charter approved by The Data Professor before work begins
- Coordinate with Sonia (PM execution) and pm-plan (sprint planning)
