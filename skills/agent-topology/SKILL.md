# Dynamic Agent Topology

Dynamically assemble optimal execution teams from requirements,
capabilities, skills, trajectories, knowledge graph, and
constitutional constraints.

## Core Principle

```
Task → Capability Analysis → Team Assembly → Provider Routing → Execution
```

## Commands

- `/araya team:recommend "<task>"` — recommend optimal team with confidence score
- `/araya team:assemble "<task>"` — assemble team with role assignments
- `/araya team:validate` — check capability coverage, skill coverage, risks
- `/araya team:risk` — detect single points of failure, overstaffing, understaffing
- `/araya team:list` — list active and recommended team formations

## Rules

- Every project must have required capability coverage
- Missing capabilities block execution
- Single points of failure must be reported
- Team recommendations must be explainable
- Topology respects constitutional rules
