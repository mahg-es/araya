# Technical Book

Write and structure technical book chapters — manuscripts, training books,
and long-form educational content — following academic and professional
publishing standards with consistent structure and pedagogical flow.

## What problem this solves
Writing a technical book without structure leads to rambling, repetition, and
reader abandonment. This skill applies book-writing methodology: chapter
structure, learning objectives, progressive complexity, and editorial standards
that produce publishable manuscripts.

## When to use
When writing technical books, book chapters, or long-form educational content.
When contributing to the Ecosystemic Enterprise Architecture book. When The
Data Professor wants to turn expertise into published work.

## Input
Topic, target audience, outline, desired chapter count, publishing format.

## Output
A structured chapter following technical book conventions with learning
objectives, body content, diagrams, code examples, and exercises.

## Steps
1. Define book structure: preface, chapters (8-15), appendices, index
2. For each chapter:
   - Learning objectives (what will the reader be able to do?)
   - Opening scenario (why does this matter?)
   - Core content: explanation → example → practice → reflection
   - Diagrams: one per major concept (use architecture-diagram skill)
   - Code examples: complete, runnable, annotated
   - Summary and exercises
3. Maintain consistent voice: authoritative but accessible, professional not academic
4. Cross-reference between chapters: "as discussed in Chapter 3..."
5. Technical review by domain specialist agent
6. Editorial review by Priscila for clarity and consistency

## Rules
- Every chapter: objectives → context → content → summary → exercises
- Diagrams for every major concept — text alone loses readers
- Code examples must be complete and runnable — no "..." or "obviously"
- Define every term on first use — no assumed knowledge
- Consistent voice across chapters — Priscila maintains editorial standards
- Citation format: author, title, year for references
- Coordinate with Priscila (editorial) and domain specialists for technical review
