# PKM Workflow

Design and optimize Personal Knowledge Management workflows — capture,
organize, retrieve, and synthesize knowledge — building systems that make
knowledge compound over time.

## What problem this solves
Information flows in constantly; without a PKM system, it flows right back out.
A PKM workflow captures valuable information, organizes it for retrieval,
connects it for insight, and synthesizes it for output — turning consumption
into creation.

## When to use
When setting up or optimizing knowledge management practices. When information
overload causes stress. When The Data Professor says "I read something about
X but can't find it now."

## Input
Current knowledge practices, tools in use, pain points (lost notes, disorganized
files, inefficient retrieval).

## Output
A PKM workflow design with capture methods, organization structure, retrieval
strategies, and synthesis practices.

## Steps
1. Map current workflow: how does information enter, get processed, and get used?
2. Identify pain points: where does information get lost or forgotten?
3. Design capture: inbox (quick notes) → process daily (daily-note)
4. Design organization: folders (top-level), tags (cross-cutting), links (connections)
5. Design retrieval: search, graph traversal, Maps of Content
6. Design synthesis: note-taking → connections → insights → output (articles, decisions)
7. Iterate: review workflow quarterly, adjust based on usage patterns

## Rules
- Capture everything potentially valuable — inbox is judgment-free
- Process daily: inbox → notes with tags, links, and summaries
- Organize by actionability, not by source — "projects/active" vs. "from book X"
- Retrieval in < 30 seconds — if you can't find it, it doesn't exist
- Synthesis is the goal: capture → organize → connect → create
- Review and iterate quarterly — PKM is personal, what works changes
