# ARAYA v0.8 — Strategic Roadmap

**Date:** 2026-05-30
**Author:** R. Daneel Olivaw
**Theme:** Context Integrity & Governance Automation
**Status:** Plan Only — No Implementation

---

## 1. Vision

ARAYA v0.7.x proved that governance can be restored after drift.
ARAYA v0.8 must **prevent drift from happening in the first place.**

The Authentik vs Authelia incident revealed the core vulnerability:
an agent (Manu) reopened a settled decision because stale context from
an old ADR leaked into a new planning cycle.

### v0.8 Theme: **Context Integrity**

```
Before (v0.7):  Detect drift after it happens → remediate
After  (v0.8):  Prevent drift before it happens → enforce
```

### Core Principle

Repository truth is not just the source of truth — it is the **only** truth.
Agents must not carry context between sessions. Every interaction starts
from a clean read of repository state.

## 3. Prioritized Backlog — v0.8

### P0 — Context Integrity (v0.8.1)

| ID | Feature | Description | Effort |
|----|---------|-------------|--------|
| CTX-01 | `/araya compact` | Produce a minimal context capsule from repository truth. Replace full agent prompts with a lean operating context. | Small |
| CTX-02 | `/araya handoff` | Generate a compact handoff document when delegating between agents. Includes: what was done, what's next, active constraints, open decisions. | Small |
| CTX-03 | ADR conflict detection in `/araya reconstitute` | Detect when a new decision conflicts with an existing ADR. Flag it before planning starts. | Small |
| CTX-04 | Session-start context reset | Every agent session starts by reading `/araya reconstitute` output. No context carried from previous sessions. | Small |

### P1 — Governance Automation (v0.8.2)

| ID | Feature | Description | Effort |
|----|---------|-------------|--------|
| GOV-01 | `/araya audit` | Independent cross-agent verification. Agent A claims X → Agent B verifies against repository. | Medium |
| GOV-02 | Run record auto-generation | Every `/araya run` and `/araya reconstitute` invocation auto-creates a run record. | Small |
| GOV-03 | Branch compliance auto-check | On session start, warn if on `main` without `dev-mahg` present. | Small |

### P2 — Simplification (v0.8.3)

| ID | Feature | Description | Effort |
|----|---------|-------------|--------|
| SIM-01 | Remove topology templates | 6 unused team templates. Keep `team:recommend` only. | Small |
| SIM-02 | Consolidate knowledge categories | 9 → 4: standards, adrs, lessons, preferences. | Small |
| SIM-03 | Remove golden trajectory activation | Framework exists but not enforced. Defer to v0.9. | Small |
| SIM-04 | Simplify AI routing | Single provider capability check. Remove multi-strategy routing. | Small |

## 6. Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Simplification breaks existing workflows | Low | Medium | Preserve all current commands; only remove unused templates |
| Context minimization loses important context | Medium | High | `/araya compact` must be reversible — full context always available |
| Agents resist clean-start model | Medium | Low | Session-start reconstitute is advisory first, mandatory later |

---

## 7. Recommended Implementation Sequence

```
v0.8.0 — Context Integrity
  /araya compact, /araya handoff, ADR conflict detection, session-start reset

v0.8.1 — Governance Automation
  /araya audit, auto run records, branch auto-check

v0.8.2 — Simplification
  Remove topology templates, consolidate knowledge, simplify routing

v0.9.0 — Learning Organization (deferred)
  Activate golden trajectories, populate knowledge graph, continuous improvement
```

## 8. What v0.8 Does NOT Do

- ❌ No new agents
- ❌ No new domains
- ❌ No knowledge graph population
- ❌ No golden trajectory activation
- ❌ No portfolio governance
- ❌ No new delivery modes

**v0.8 is about making what exists work reliably.** No expansion. Only integrity.
