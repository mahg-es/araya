# docs/commands — AX3 Local Contract

## Purpose

Command reference documentation for the ARAYA Framework. Each document covers one command with: purpose, syntax, all modes, expected output samples, and troubleshooting.

## Ownership

Priscila (Technical Writer) — documentation.
Valentina (Backend Developer) — technical review of command behavior.

## Local Contracts

- Command references must be validatable against the canonical catalog
- Every document must carry: date, author, version, status
- Code blocks must specify language and be copy-paste runnable
- All command flags and arguments must be documented

## Work Guidance

- Command reference template: What → Why → Modes → Output Samples → Troubleshooting → Related
- Coordinate with the catalog maintainer (Valentina) for accuracy
- Use output samples from the live help-provider, not invented examples
